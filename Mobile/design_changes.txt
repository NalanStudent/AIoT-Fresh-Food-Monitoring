Mobile App Design Specification: AIoT Fresh Monitor

Objective: Create a high-fidelity mobile application interface for an IoT logistics monitoring system. The design must support a "Professional Navy & Gold" theme with fully integrated Light and Dark modes.

1. Visual Style & Theme

Typography:

Font Family: 'Inter', sans-serif (Clean, modern, legible).

Weights: 400 (Regular), 600 (Semi-Bold), 700 (Bold).

Color Palette:

Primary Action/Highlight: Golden Yellow (#FFC107) - Used for active tab icons, key data values, and interactive highlights.

Primary Brand: Deep Navy (#00334E) - Used for user chat bubbles, light mode headers, and light mode active states.

Dark Mode Backgrounds:

App Background: Deep Charcoal (#141b1f)

Card Background: Dark Slate (#1e282e)

Text: Off-white (#E0E0E0)

Light Mode Backgrounds:

App Background: Pale Gray (#F5F7FA)

Card Background: Pure White (#FFFFFF)

Text: Dark Slate (#263238)

Status Indicators:

Success/Online: Forest Green (#2E7D32)

Critical/Alert: Crimson Red (#C62828)

Warning: Orange (#EF6C00)

Global UI Elements:

Corner Radius: 20px for the main screen container; 12px for internal cards and buttons.

Shadows: Soft, diffused shadows to create depth (lighter in Light mode, deeper in Dark mode).

Transitions: Smooth transitions (0.3s) for background colors when switching themes.

2. Navigation Structure

Bottom Navigation Bar:

Fixed at the bottom of the screen.

3 Tabs:

Home (Icon: House) - Active state highlights icon and label in Gold (Dark Mode) or Navy (Light Mode).

AI Assist (Icon: Robot Face)

Settings (Icon: Gear)

Background matches the card background color of the active theme.

3. Screen Specifications

A. Home Screen (Dashboard)

Header:

Left: App Title "AIoT Fresh Monitor" (Bold).

Right: Cloud Emoji or Icon.

Content: Scrollable list of container cards.

Container Card Design:

Header Row: Container ID (Left) and Status Badge (Right).

Status Badge: Colored dot + Text (e.g., Green Dot + "Online").

Info Row: Cargo type (e.g., "Bananas").

Telemetry Row: Horizontal layout of key metrics with icons (e.g., üå°Ô∏è 12.5¬∞C, üíß 90%).

Interaction: Tapping a card opens the Device Detail Screen.

Offline State: Reduce opacity (0.8) and display a "CRITICAL ALERT" banner at the bottom of the card in Red.

B. Device Detail Screen (Drill-down view)

Header:

Left: Back Arrow (<-) to return to Dashboard.

Center: Container ID.

Status Card: Summary of Food Type, Connection Status, and Last Seen timestamp.

Telemetry Grid:

2x2 Grid Layout.

Each box contains a Label (small, uppercase) and a Value (large, colored).

Contextual Coloring: Values should reflect their status (e.g., High Temp = Red, Normal = Green).

Map Placeholder: A dashed-border box indicating geospatial location.

Alerts List: A chronological list of recent alerts (Critical/Warning), styled with colored dots and timestamps.

C. AI Assist Screen

Header: Title "AI Assistant" with a Sparkle icon.

Chat Interface:

Chat History area: Scrollable.

AI Bubbles: Left-aligned, dark gray (Dark mode) or light gray (Light mode) background.

User Bubbles: Right-aligned, Navy Blue background, White text.

Prompt Chips: Horizontal scrollable list of quick actions (e.g., "Check status", "Forecast temp") located above the input bar.

Input Area: Text field with rounded corners and a Send button (Arrow icon).

D. Settings Screen

Header: Title "Settings" with a Gear icon.

Section 1: Appearance

Dark Mode Toggle: A switch control to instantly invert the application's color theme between Light and Dark modes.

Section 2: Command Settings

List of toggle switches for device controls (e.g., "Remote Reboot", "Firmware Auto-Update").

Read-only settings displayed as text (e.g., "Polling Interval: 10s").

Section 3: Notifications

Toggle for "Push Alerts".
